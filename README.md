\# Question Answer



\#Q1. What is the difference between null and undefined?
- Ans: undefined is for value where no value is present. But null is for value where value is not yet present but in the future it will/may be added.



\#Q2. What is the use of the map() function in JavaScript? How is it different from forEach()?
- Ans: map() is used to return an array after processing a given array. forEach() is used to perform some task on an array but doesn't return anything.



\#Q3. What is the difference between == and ===?
- Ans: == check only the value but === checks both value and data typs. In JS different types of data show same result after implicite data conversion. So it is better to use === instead of == for desire output.



\#Q4. What is the significance of async/await in fetching API data?
- Ans: To force the code to run in an asynchronous way async/await is used. As fetching API returns a Promise, an asynchronous response, async/await is used to wait till the Promise is resolve or reject.



\#Q5. Explain the concept of Scope in JavaScript (Global, Function, Block).
- Ans: Access area of a variable or data is its scope. For global scope, any value is accessable from anywhere after TDZ ends. For Functional or Block scope, variable or data can't be accessed outside that function or block.
